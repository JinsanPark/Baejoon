### 문제 설명

<p>아래 그림처럼 높이만 다르고 (같은 높이의 막대기가 있을 수 있음) 모양이 같은 막대기를 일렬로 세운 후, 왼쪽부터 차례로 번호를 붙인다. 각 막대기의 높이는 그림에서 보인 것처럼 순서대로 6, 9, 7, 6, 4, 6 이다. 일렬로 세워진 막대기를 오른쪽에서 보면 보이는 막대기가 있고 보이지 않는 막대기가 있다. 즉, 지금 보이는 막대기보다 뒤에 있고 높이가 높은 것이 보이게 된다. 예를 들어, 그림과 같은 경우엔 3개(6번, 3번, 2번)의 막대기가 보인다.</p>

<p>N개의 막대기에 대한 높이 정보가 주어질 때, 오른쪽에서 보아서 몇 개가 보이는지를 알아내는 프로그램을 작성하려고 한다.</p>

### 입력 

 <p>첫 번째 줄에는 막대기의 개수를 나타내는 정수 N (2 ≤ N ≤ 100,000)이 주어지고 이어지는 N줄 각각에는 막대기의 높이를 나타내는 정수 h(1 ≤ h ≤ 100,000)가 주어진다.</p>

### 출력 

 <p>오른쪽에서 N개의 막대기를 보았을 때, 보이는 막대기의 개수를 출력한다.</p>

## 1. 초기 접근 방식
-초기 접근: '오른쪽에서 볼 때 몇 개가 보이는가'를 묻는 문제이므로, 데이터의 오른쪽부터 하나씩 꺼내어 확인하기 위해 스택(Stack) 구조를 떠올림.
-로직 구상: pop() 한 값을 저장해 두고, 그 다음에 pop() 한 값이 앞서 저장된 값보다 크면 보이는 것이므로 count++를 하는 방식으로 접근함.
- 문제 풀이 중 추가한 점: 단순히 바로 직전에 꺼낸 값과 비교하는 것만으로는 부족함을 깨닫고, 기둥의 max값(현재까지 본 가장 높은 기둥의 높이)을 추적하는 변수를 추가함.

## 2. AI 튜터 활용

질문 : '기존의 Deque를 사용한 로직보다 더 빠르고 효율적으로는 어떻게 할까?'라고 질문.

AI의 답변 :

1. 고정된 크기(N)와 기본 자료형(Primitive Type) 활용하기
 Deque<Integer>나 ArrayList<Integer>는 내부적으로 int를 Integer 객체로 변환하는 박싱/언박싱 과정을 거치며 추가 메모리를 사용합니다. 문제에서 N의 크기가 주어지므로, int[] 배열을 사용하면 메모리 사용량을 줄이고 접근 속도를 극대화할 수 있습니다.

2. 배열의 역순 순회(Backward Traversal) 제어
스택의 LIFO 원리를 배열로 구현하려면 인덱스를 끝에서부터 0까지 줄여나가야 합니다. 이때 배열의 마지막 인덱스는 N이 아닌 N-1이며, 0번 인덱스까지 포함하도록 i >= 0 조건을 설정하는 것이 중요합니다.

얻은 깨달음 :
반드시 Stack이나 Deque 라이브러리를 쓰지 않아도 배열로도 동일한 효과를 더 빠르게 낼 수 있음.
